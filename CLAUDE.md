# Garden — Focus Timer

Веб-приложение для фокусировки в духе Forest. Завершённые сессии «сажают» деревья
в изометрическом саду 5×5. Есть таймер, экономика монет, магазин, статистика.

## Главное

- **Отвечай по-русски.**
- Это пет-проект начинающего разработчика. Объясняй изменения простыми словами.
- **Точность важнее похвалы.** Не пиши, что фича готова, если её нет в коде.
  Описывай реальное состояние файлов, а не то, что «обсуждалось».

  файлов целиком («second system effect» — плохо).
- Когда есть два способа — предлагай более простой.

## Стек и структура

Фронтенд — чистые HTML/CSS/JS, без фреймворков. Бэкенд — Flask + SQLite.

| Файл         | Что внутри                                                        |
|--------------|-------------------------------------------------------------------|
| `index.html` | Разметка: таймер, сад, статистика, попапы                         |
| `style.css`  | Все стили + переменные тем (`:root` и `body.dark`)                |
| `app.js`     | Вся логика: таймер, сад, магазин, статистика, хитмап              |
| `trees.js`   | Константы, `TAGS`, каталог `TREES`, `getDefaultUnlocked()`        |
| `lands.js`   | Каталог `LANDS`, `getDefaultUnlockedLands()`                      |
| `quotes.js`  | Массив цитат `QUOTES`                                             |
| `app.py`     | Flask-сервер: отдаёт статику, `GET/POST /api/state`               |
| `garden.db`  | SQLite, одна таблица `state` (key='garden', value=JSON)           |
| `garden.bat` | Лаунчер: запускает Flask свёрнутым и открывает браузер            |

Скрипты подключаются в `index.html` в порядке: `trees.js` → `lands.js` →
`quotes.js` → `app.js`. Константы из `trees.js`/`lands.js` доступны в `app.js`.

Спрайты: деревья в `/sprites/` (формат `{type}-{stage}.png`, stage 1–3),
тайлы земли в `/land/` (`{key}.png`), бейджи достижений в `/badges/`.
Пайплайн ассетов: генерация картинки → remove.bg → раскладка по папкам.

## Реализованные фичи

Таймер с SVG-циферблатом, тегами и журналом сессий; изометрический сад со
спрайтами и сортировкой по глубине (painter's algorithm); история садов по
датам с навигацией `‹ ›`; тайлы земли; экономика монет и магазин (деревья +
земля) с подтверждением покупки; статистика (вкладки периодов, стрик,
аналитические карточки, годовой хитмап в стиле GitHub, недельный паттерн,
распределение по тегам/деревьям); **система достижений** (бейджи в `/badges/`,
попап со списком + попап-празднование при разблокировке); фоновые эффекты
(облака / звёзды / снег по типу земли); цитаты во время сессий; экспорт/импорт
JSON в настройках.

## Как запускать

Двойной клик по `garden.bat`, либо вручную:

```
python app.py
```

Flask поднимается на `http://127.0.0.1:5000` (debug=True). Держи отдельную
вкладку терминала под сервер, пока работаешь.

## Хранение данных

Всё состояние (`state`) хранится на бэкенде через `/api/state` (не localStorage!).
`loadState()` тянет JSON при старте, `saveState()` шлёт POST после каждой сессии.
Единственное, что осталось в localStorage — выбор темы (`forest_theme`).

Структура `state`: `coins`, `sessions[]`, `gardens{}` (по датам), `recentPlants[]`,
`activeLand`, `unlocked.{trees,lands}`. Версия схемы — `STATE_VERSION = 3`,
миграция в `migrate()`.

## Грабли, которые уже наступали (НЕ повторять)

- **Даты только в локальном времени.** `getToday()` использует `getFullYear/
  getMonth/getDate`, НЕ `toISOString()` — иначе сад сбрасывается в 2 ночи (UTC+2).
- **Дату сессии фиксируй на старте**, а не на финише (`sessionStartDate`),
  иначе сессии через полночь попадают не в тот день.
- **Не накапливай слушатели.** Для перерисовываемых блоков используй
  `onclick =` или паттерн `cloneNode` + `replaceChild` (как в `renderTagRow`,
  `showConfirm`), а не повторный `addEventListener`.
- **SVG статистики рендерится в ~2× из-за viewBox vs контейнер.** В графиках
  `font-size` около 6.0–6.5 SVG-единиц — это правильно. Десятичный разделитель —
  только точка, не запятая.
- **Схлопнутые секции:** `margin-top` остаётся даже при `.collapsed` — нужен
  явный `margin-top: 0` в классе `.collapsed`.
- Проверяй дубли блоков (Ctrl+F) и тестируй каждую правку перед следующей.
- Следи за случайной кириллицей в CSS/коде и путями к файлам.
